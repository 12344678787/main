    Assign a Role Collection to a User

    Documentation

    • Security Administration: Managing Authentication and Authorization (SAP Help)

    To be able to access the application, your user needs to be assigned to a role collection that provides the requires scopes.

    1. Open the SAP BTP Cockpit.
    2. Go to the Subaccount.
    3. Choose Trust Configuration under Security on the left.
    4. Choose Default identity provider.
    5. Enter the E-Mail Address of your user.
    6. Choose Show Assignments.
    7. Choose Assign Role Collection.
    8. Select RiskManager in dropdown Role Collection.
    9. Choose Assign Role Collection.

    The role collection RiskManager appears now in the list of your user's role collections.

    Test It

    Go to the application and choose the tile Risks. You should be able to see and change the data in the Risks application.

    If not, probably you've signed in before adding the role collection to your user. Sign out and sign in again to get the added roles in your login ticket.

    Additional Info: Create a Role Collection Manually

    Instead of creating the role collection automatically using the xs-security.json, it's also possible to create a role collection manually. For example:

    1. Open the SAP BTP Cockpit.
    2. Go to the Subaccount.
    3. Choose Role Collections.
    4. Choose the + (new role collection) button.
    5. Enter Name RiskManager.
    6. Choose Create.

    The new role collection appears now in the list, but it doesn't contain any roles. To add a role:

    1. Choose the RiskManager role collection.
    2. Choose Edit.

    3. Open the value help for Role Name

    4. Select cpapp-...!... in Application Identifier dropdown.

      The App Identifier is the XSAPPID of your application. It consists of the XSAPPNAME that has been used to create the xsuaa service followed by an exclamation mark (!) and a landscape-wide unique ID.

    5. Select RiskManager in list of roles.

    6. Choose Add.
    7. Choose Save.
