    Undeploy Your Cloud Foundry Application

    Do the following steps if you want to undeploy your application, depending on how you did the deployment:

    Option 1: Undeploy an Application Deployed with cf push

    You need to delete all created applications and services. First delete the applications, which will delete their service bindings as well, then you can delete the services.

    These are the steps to undeploy the application from the Deploy Your Application Using cf push tutorial module:

    1. Delete applications:

      cf delete cpapp-app -f
cf delete cpapp-srv -f
cf delete cpapp-db-deployer -f

    2. Delete service instances:

      Deleting the cpapp-db service instance deletes all data stored in the application.

      cf delete-service cpapp-db -f
cf delete-service cpapp-uaa -f

    Option 2: Undeploy a MultiApps Application

    To undeploy (delete) an MTAR, you don't need to delete the apps and the services individually. The deploy service on Cloud Foundry keeps track on the deployed mtar and its resources.

    You can either undeploy the applications or undeploy the applications and its service instances, which will delete the data stored in these instances (for example, database content) as well.

    • Option A: Delete applications only:

      cf undeploy cpapp

    • Option B: Delete applications and service instances:

      Deletes all data stored in the application.

      cf undeploy cpapp --delete-service-keys --delete-services
